# Smart FinTech Assistant

## Project Overview

**Smart FinTech Assistant** is a professional-grade, AI-powered financial learning and paper trading platform. It provides users with a safe, simulated environment to master financial markets through hands-on practice and intelligent guidance.

With this platform, users can:
- **Learn Financial Concepts**: Access structured lessons and interactive quizzes.
- **Simulate Stock Trading**: Execute real-time paper trades using live market data.
- **Analyze Portfolio Performance**: Visualize holdings, allocation, and historical performance using high-fidelity charts.
- **Receive AI-Generated Insights**: Get personalized market commentary and strategy advice powered by Google Gemini.

---

## Features

- **Paper Trading Simulator**: Real-time order execution for Indian Stocks, US Stocks, and Cryptocurrencies.
- **Portfolio Analytics Dashboard**: Comprehensive breakdown of ROI, P&L, and asset allocation using Recharts.
- **AI Financial Insights**: Context-aware portfolio analysis and market sentiment generated by Gemini 2.5 Flash.
- **Financial Learning Module**: Interactive lessons and quizzes designed for beginner investors.
- **Professional Trading UI**: A premium, TradingView-style dark terminal interface designed for institutional-grade UX.

---

## Technology Stack

- **Frontend**: React, Vite, TailwindCSS, Recharts, Lightweight Charts.
- **Backend**: FastAPI (Python), Uvicorn.
- **AI**: Gemini API (`google-generativeai`).
- **Cache/Storage**: Redis (with automatic In-Memory fallback).

## Quick Start Guide

Follow these steps to get the project running locally on your machine.

### Step 1 — Clone the Repository
```bash
git clone https://github.com/<username>/Smart-FinTech-Assistant.git
cd Smart-FinTech-Assistant
```

### Step 2 — Backend Setup
1. Navigate to the backend folder:
   ```bash
   cd backend
   ```
2. Create and activate a virtual environment:
   ```bash
   python -m venv venv
   # Windows:
   venv\Scripts\activate
   # Linux / Mac:
   source venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

### Step 3 — Configure Environment Variables
Copy the example file from the project root and add your details:
```bash
# Return to root
cd ..
cp .env.example .env
```
Edit `.env` and configure your keys:
- `GEMINI_API_KEY`: Your key from [Google AI Studio](https://aistudio.google.com/).
- `GEMINI_MODEL`: `models/gemini-2.5-flash`
- `REDIS_URL`: `redis://localhost:6379` (Optional; system will fallback to memory).
- `SECRET_KEY`: Any secure random string.

### Step 4 — Start Backend Server
```bash
cd backend
uvicorn main:app --reload
```
The backend will be available at: **http://localhost:8000**

### Step 5 — Frontend Setup
1. Open a new terminal and navigate to the frontend folder:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Start the development server:
   ```bash
   npm run dev
   ```
The frontend will be available at: **http://localhost:5173**

## API Overview

- `/api/auth`: User registration, login, and session management.
- `/api/portfolio`: Current holdings and time-series performance data.
- `/api/trade`: Order execution (BUY/SELL) and transaction history.
- `/api/ai`: Chat with the terminal advisor and get portfolio/market insights.
- `/api/ai/debug-models`: **(Diagnostic)** Lists available Gemini models and tests connectivity.

---

## Troubleshooting

- **Gemini Model Error (404/400)**: Ensure `GEMINI_MODEL=models/gemini-2.5-flash` is set in your `.env`.
- **Environment Not Loading**: Restart the backend server after making any changes to `.env`.
- **Redis Connection Error**: This is harmless; the system will automatically switch to "In-Memory" mode for local development.

---

## Security Notes

- **Never commit your `.env` file**. It contains sensitive API keys.
- Ensure the `SECRET_KEY` in production is a long, random string to protect user sessions.
- Use `.env.example` as a template for adding new environment variables.
